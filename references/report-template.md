## 填写指南

### 各Section要点

| Section | 核心要点 | 字数建议 |
|---------|---------|---------|
| 一、问题概述 | 简洁说明问题和状态 | 100-150字 |
| 二、事实回顾 | 时间线+沟通记录 | 200-300字 |
| 三、原因分析 | 5Why链条 | 200-300字 |
| 四、决策回溯 | 关键决策评估 | 100-200字 |
| 五、已采取行动 | 当前补救措施 | 100-150字 |
| 六、改进措施 | 具体措施+责任人+时间 | 300-400字 |
| 七、总结反思 | 核心教训 | 100-150字 |

### 语气要求

| 要求 | 说明 |
|------|------|
| **客观** | 基于事实，不推诿 |
| **具体** | 用具体事例，不用模糊词 |
| **行动导向** | 重点在改进措施 |
| **系统性** | 强调机制改进 |
| **简洁** | 每section控制在建议字数 |

---

## 常见错误

### 需避免的写法

| 错误写法 | 问题 | 正确写法 |
|---------|------|---------|
| 「差距很大」 | 模糊 | 「功能完整度约60%，缺失混剪能力」 |
| 「以后要注意」 | 口号 | 「建立承诺前置审批流程」 |
| 「某人失误导致」 | 追责导向 | 「评估流程缺失，导致识别遗漏」 |
| 「运气不好」 | 不可控因素 | 「风险预警机制缺失，未能预判变化」 |

## 报告自检清单

提交前检查：

```
□ 问题概述是否简洁（≤150字）？
□ 事实是否客观，观点是否标注？
□ 5Why是否挖到系统性原因？
□ 决策评估是否具体？
□ 改进措施是否有责任人+时间？
□ SOP是否可执行？
□ 总字数是否≤1500字？
□ 是否有不推诿的语气？
```
