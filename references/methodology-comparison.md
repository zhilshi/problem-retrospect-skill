## 二、方法论详解

### 1. 丹纳赫5Why分析法（已整合到problem-retrospect技能）

**起源**：丹纳赫公司（Danaher Corporation），制造业持续改进标杆

**核心方法**：
```
Why 1 → Why 2 → Why 3 → Why 4 → Why 5（系统性根因）
```

**关键特点**：
- 穿透表象，不停在个人失误层面
- 必须挖到流程/机制/认知层面才算完成
- 90%的问题来自流程，10%来自人员

**适用场景**：
- 复杂问题的根因分析
- 重复性问题深挖
- 向上汇报的复盘报告

**优点**：结构清晰、穿透力强、指向系统性改进
**缺点**：需要足够信息支撑、主观判断重

---

### 2. AAR（After Action Review，事后回顾）

**起源**：美国陆军，军事行动后的标准复盘流程

**核心方法**：4个标准问题

| # | 问题 | 目的 |
|---|------|------|
| 1 | 我们预期发生什么？ | 建立预期基准 |
| 2 | 实际发生了什么？ | 建立事实基础 |
| 3 | 为什么会有差异？ | 分析差距原因 |
| 4 | 我们学到了什么？ | 提取经验教训 |

**关键特点**：
- 时间短（15-30分钟）
- 非正式、无责备氛围
- 所有参与者平等发言
- 立即记录、立即分享

**适用场景**：
- 任务/项目完成后快速回顾
- 团队日常复盘
- 紧急事件后快速总结

**优点**：快速、简单、全员参与
**缺点**：不够深入、根因分析弱

---

### 3. KPT（Keep/Problem/Try）

**起源**：敏捷软件开发

**核心方法**：三分类反思框架

| 分类 | 问题 | 焦点 |
|------|------|------|
| **Keep** | 什么值得保持？ | 有效做法 |
| **Problem** | 遇到什么问题？ | 需改进点 |
| **Try** | 下次尝试什么？ | 新行动 |

**关键特点**：
- 正负平衡（保持+问题+尝试）
- 行动导向（每项都要转化为下次行动）
- 可视化（白板/便利贴）

**适用场景**：
- 敏捷迭代复盘（Sprint Retro）
- 个人工作反思
- 小团队周复盘

**优点**：简单易用、行动导向、平衡正负
**缺点**：根因分析弱、适合小问题

---

### 4. Start/Stop/Continue

**起源**：敏捷开发，比KPT更行为导向

**核心方法**：三个行为问题

| 分类 | 问题 | 焦点 |
|------|------|------|
| **Start** | 应该开始做什么？ | 新增行为 |
| **Stop** | 应该停止做什么？ | 消除行为 |
| **Continue** | 应该继续做什么？ | 保持行为 |

**关键特点**：
- 比KPT更行为导向
- 明确的行动指令（开始/停止/继续）
- 易于跟踪执行

**适用场景**：
- 团队行为规范调整
- 流程改进
- 个人习惯改进

**优点**：行动明确、易于跟踪
**缺点**：偏向行为层面、根因弱

---

### 5. Starfish（海星复盘）

**起源**：Patrick Kua开发，扩展Start/Stop/Continue

**核心方法**：五维度反思（像海星五个角）

| 维度 | 问题 | 焦点 |
|------|------|------|
| **Keep** | 保持做什么？ | 有效做法 |
| **More** | 增加做什么？ | 强化行为 |
| **Less** | 减少做什么？ | 减弱行为 |
| **Stop** | 停止做什么？ | 消除行为 |
| **Start** | 开始做什么？ | 新增行为 |

**关键特点**：
- 增加「程度调节」（More/Less）
- 更精细的行为控制
- 适合调整工作强度

**适用场景**：
- 团队工作量调整
- 流程强度优化
- 避免「一刀切」的改进

**优点**：程度可调节、更精细
**缺点**：维度多可能分散焦点

---

### 6. PDCA（Plan/Do/Check/Act）

**起源**：戴明环（Deming Cycle），质量管理经典

**核心方法**：四步循环

| 步骤 | 动作 | 内容 |
|------|------|------|
| **Plan** | 计划 | 设定目标、制定计划 |
| **Do** | 执行 | 执行计划、记录数据 |
| **Check** | 检查 | 对比结果、分析差距 |
| **Act** | 改进 | 标准化成功做法、改进失败点 |

**关键特点**：
- 循环迭代（Check→Act→Plan...）
- 数据驱动
- 标准化成功做法

**适用场景**：
- 持续改进流程
- 质量管理
- 流程优化

**优点**：循环迭代、标准化、数据驱动
**缺点**：需要周期性执行、不适合单次复盘

---

### 7. A3（丰田一页纸问题解决）

**起源**：丰田汽车，精益制造核心工具

**核心方法**：一页A3纸结构化报告

| Section | 内容 |
|---------|------|
| **背景** | 为什么这个问题重要？ |
| **现状** | 当前情况数据 |
| **目标** | 期望达成的状态 |
| **根因分析** | 5Why分析 |
| **对策** | 解决方案 |
| **效果确认** | 实施效果 |
| **后续行动** | 标准化/推广 |

**关键特点**：
- 一页纸强制精炼
- 数据可视化
- 需要多轮迭代
- 现场观察（Gemba）

**适用场景**：
- 系统性问题解决
- 跨部门问题协调
- 需向上汇报的复杂问题

**优点**：结构完整、可视化、精炼
**缺点**：时间投入大、需要数据支撑

---

### 8. DMAIC（六西格玛改进流程）

**起源**：六西格玛（Six Sigma），摩托罗拉/GE

**核心方法**：五步数据驱动改进

| 步骤 | 动作 | 内容 |
|------|------|------|
| **Define** | 定义 | 明确问题、范围、目标 |
| **Measure** | 测量 | 收集数据、建立基线 |
| **Analyze** | 分析 | 找根因、验证假设 |
| **Improve** | 改进 | 设计方案、实施验证 |
| **Control** | 控制 | 标准化、监控维持 |

**关键特点**：
- 数据驱动决策
- 统计分析工具
- 严格验证假设
- 长期监控控制

**适用场景**：
- 复杂质量问题
- 高风险流程改进
- 需精确数据的场景

**优点**：严谨、数据驱动、可量化
**缺点**：需要专业统计知识、时间长

---

### 9. 8D（Eight Disciplines，八步问题解决）

**起源**：福特汽车，汽车行业标准

**核心方法**：八步团队问题解决

| # | 步骤 | 内容 |
|---|------|------|
| D0 | 准备 | 组建团队、紧急措施 |
| D1 | 团队 | 组建跨职能团队 |
| D2 | 描述 | 量化描述问题 |
| D3 | 临时措施 | 防止问题扩大 |
| D4 | 根因 | 找到真正原因 |
| D5 | 永久措施 | 设计根本解决方案 |
| D6 | 实施 | 执行并验证 |
| D7 | 预防 | 防止再发生 |
| D8 | 认可 | 团队表彰 |

**关键特点**：
- 强调团队协作
- 临时措施+永久措施分离
- 防止再发生机制

**适用场景**：
- 跨部门复杂问题
- 质量事故
- 客户投诉处理

**优点**：流程完整、团队协作、防复发
**缺点**：步骤多、时间长

---

### 10. ORID（Objective/Reflective/Interpretive/Decisional）

**起源**：ICA（Institute of Cultural Affairs），焦点对话法

**核心方法**：四层对话结构

| 层级 | 问题类型 | 示例问题 |
|------|---------|---------|
| **O** 客观层 | 看到什么事实？ | 「发生了什么？」 |
| **R** 反应层 | 感受如何？ | 「你当时什么感受？」 |
| **I** 解释层 | 意义是什么？ | 「这意味着什么？」 |
| **D** 决定层 | 要做什么？ | 「我们接下来怎么做？」 |

**关键特点**：
- 从事实→感受→意义→行动的递进
- 包含情感层面
- 引导深度思考

**适用场景**：
- 深度反思对话
- 情感化事件复盘
- 团队冲突处理后的反思

**优点**：包含情感、结构递进、深度反思
**缺点**：时间较长、需要引导技巧

---

### 11. Toyota Kata（丰田套路）

**起源**：丰田，Mike Rother系统总结

**核心方法**：改进套路 + 教练套路

**改进套路（Improvement Kata）**：

| 步骤 | 内容 |
|------|------|
| 1 | 理解方向（愿景） |
| 2 | 掌握现状 |
| 3 | 设定下一目标状态 |
| 4 | 向目标迭代实验 |

**教练套路（Coaching Kata）**：
- 5个教练问题引导改进
- 不给答案，引导思考

**关键特点**：
- 不只是解决问题，是建立改进习惯
- 小步实验，快速迭代
- 教练引导，不直接给答案

**适用场景**：
- 组织能力建设
- 长期持续改进文化
- 团队习惯养成

**优点**：培养习惯、能力建设、可持续
**缺点**：需要长期投入、教练培养

---

### 12. Appreciative Inquiry（欣赏式探询）

**起源**：David Cooperrider，正向变革方法

**核心方法**：4D循环（发现→梦想→设计→命运）

| 步骤 | 动作 | 焦点 |
|------|------|------|
| **Discover** | 发现 | 发现成功案例、优势 |
| **Dream** | 梦想 | 想象理想未来 |
| **Design** | 设计 | 设计实现路径 |
| **Destiny** | 命运 | 执行、迭代 |

**关键特点**：
- 正向聚焦（发现成功而非问题）
- 激励导向（梦想理想状态）
- 不找问题，找优势放大

**适用场景**：
- 组织变革
- 团队士气提升
- 战略规划

**优点**：正向激励、团队参与度高
**缺点**：不适合明确问题根因分析

## 三、选择决策矩阵

### 按问题复杂度选择

| 复杂度 | 推荐方法 | 备选 |
|-------|---------|------|
| **低**（日常小事） | KPT、Start/Stop/Continue | AAR |
| **中**（需要分析原因） | 5Why、AAR | ORID、Starfish |
| **高**（系统性问题） | A3、8D、DMAIC | 5Why深度版 |
| **长期**（能力建设） | Toyota Kata、PDCA循环 | Appreciative Inquiry |

### 按场景类型选择

| 场景类型 | 推荐方法 | 原因 |
|---------|---------|------|
| **敏捷迭代复盘** | KPT、Starfish | 快速、行动导向 |
| **项目交付复盘** | AAR、5Why | 结构化、根因分析 |
| **质量问题** | 8D、DMAIC | 数据驱动、防止再发 |
| **跨部门协作问题** | A3、8D | 团队协作、可视化 |
| **向上汇报** | 5Why、A3 | 结构清晰、指向改进 |
| **团队士气/文化** | Appreciative Inquiry | 正向激励 |
| **情感化事件** | ORID | 包含情感层面 |
| **长期改进文化** | Toyota Kata | 习惯养成 |

### 按时间投入选择

| 时间 | 推荐方法 |
|------|---------|
| **15-30min** | AAR、KPT、Start/Stop/Continue |
| **30-60min** | 5Why、Starfish、ORID |
| **1-2h** | AAR深度版、PDCA单轮 |
| **1-2天** | A3 |
| **1-4周** | DMAIC、8D |
| **长期** | Toyota Kata、PDCA循环 |

## 四、方法论组合使用

### 常见组合模式

| 组合 | 场景 | 执行方式 |
|------|------|---------|
| **AAR + 5Why** | 项目复盘 | AAR快速回顾 → 发现问题用5Why深挖 |
| **KPT + PDCA** | 敏捷改进 | KPT发现问题 → PDCA持续改进 |
| **Starfish + ORID** | 团队调整 | Starfish行为调整 → ORID深度反思 |
| **A3 + Toyota Kata** | 系统改进 | A3解决具体问题 → Kata培养习惯 |
| **DMAIC + 8D** | 重大质量事故 | 8D快速响应 → DMAIC长期根治 |

### 组合原则

1. **快速方法发现问题，深度方法分析根因**
2. **行为方法调整做法，结构方法系统解决**
3. **正向方法激励团队，分析方法解决问题**

## 五、整合建议：扩展problem-retrospect技能

### 建议增加的方法论选项

| 方法论 | 整合方式 | 优先级 |
|-------|---------|--------|
| **AAR** | 作为Phase 1的快速采集框架 | 高 |
| **ORID** | 作为Phase 3.5的情感层面补充 | 中 |
| **Starfish** | 作为Phase 4的行为改进框架 | 中 |
| **PDCA** | 作为Phase 4后的持续跟踪框架 | 中 |
| **A3** | 作为Phase 5的可视化报告格式备选 | 高 |

### 整合后的技能升级路径

```
Phase 1: 事实采集
  → 默认：4Agent并行（时间线/决策链/沟通/资源）
  → 快速模式：AAR 4问框架

Phase 3: 根因分析
  → 默认：5Why
  → 备选：8D D4步骤

Phase 4: 方案制定
  → 默认：改进三维度（即时/机制/能力）
  → 备选：Start/Stop/Continue（行为导向）
  → 备选：Starfish（强度调节）

Phase 5: 报告生成
  → 默认：向上汇报文档
  → 备选：A3一页纸可视化报告
```
